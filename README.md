# mitm_capcha_killer

基于 mitmproxy 的本地验证码自动识别代理。通过在请求中插入标记关键字，代理自动拦截请求、获取验证码图片、OCR 识别并回填结果，实现验证码的全自动处理。

## 工作原理

```
客户端请求 (含 @cap@1@ 标记)
       ↓
  mitmproxy 拦截 (端口 8081)
       ↓
  匹配配置 profile 1
       ↓
  请求验证码图片 → OCR 识别
       ↓
  替换标记为识别结果 → 转发请求
```

1. 在需要填入验证码的请求参数中，将验证码值替换为 `@cap@{ID}@`（ID 对应配置编号）
2. 代理拦截到含标记的请求后，根据对应配置获取验证码图片并执行 OCR 识别
3. 将标记替换为识别结果后转发原始请求
4. 高级模式下，还可通过正则从验证码响应中提取 Token 等数据，使用 `@cap@x{ID}@` 标记回填

## 环境要求

- Python >= 3.10
- Windows / Linux / macOS

## 安装

```bash
# 创建虚拟环境并安装依赖
uv venv
uv sync
```

依赖项：
- `ddddocr` — OCR 识别引擎
- `mitmproxy` — HTTP/HTTPS 代理
- `requests` — HTTP 请求库

## 运行

```bash
python main.py
```

启动后：
- **代理服务**：`0.0.0.0:8081`（mitmproxy）
- **Web 管理面板**：`http://127.0.0.1:8899/`

## 关键字说明

| 关键字 | 用途 | 示例 |
|--------|------|------|
| `@cap@{ID}@` | OCR 识别结果占位符 | `@cap@1@` → 识别出的验证码文本 |
| `@cap@x{ID}@` | 正则提取结果占位符（高级模式） | `@cap@x1@` → 正则提取的 Token/Session |

将关键字放置在请求体或请求头中需要填入验证码/Token 的位置即可。

## 配置说明

通过 Web 面板或直接编辑 `config.json` 进行配置：

```json
{
  "switchs": 1,
  "whitelist_switch": 0,
  "whitelist_hosts": [],
  "profiles": {
    "1": ["url", "type", "charset", "raw_request", "extract_from", "regex", "advanced"]
  }
}
```

### Profile 字段

| 索引 | 字段 | 说明 |
|------|------|------|
| 0 | 目标 URL | 验证码图片请求地址 |
| 1 | 请求模式 | `1` = GET，`2` = 自定义原始请求 |
| 2 | 输出模式 | 字符集过滤：`0` 数字 / `1` 小写 / `6` 混合 / `8` 跳过 OCR |
| 3 | 原始请求 | 模式 2 时的完整 HTTP 请求报文 |
| 4 | 提取来源 | `0` = 响应体，`1` = 响应头 |
| 5 | 正则表达式 | 高级模式下的提取规则 |
| 6 | 高级开关 | `"true"` / `"false"` |

### 输出模式（字符集）

| 值 | 范围 |
|----|------|
| 0 | 纯数字 |
| 1 | 小写字母 |
| 2 | 大写字母 |
| 3 | 大小写字母 |
| 4 | 小写字母 + 数字 |
| 5 | 大写字母 + 数字 |
| 6 | 大小写字母 + 数字（默认） |
| 8 | 跳过 OCR（仅用正则提取） |

## Web 管理面板

面板提供以下功能：

- **全局设置**：白名单模式开关及主机配置
- **配置管理**：添加/删除/编辑验证码识别配置（最多 10 个）
- **代理控制**：启动/停止代理服务
- **实时监控**：运行日志 + 最近 20 条识别记录（含验证码图片预览）
- **正则帮助**：内置正则表达式书写参考
